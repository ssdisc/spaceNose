## 🔌 一、硬件连接方案

### 1.1 MQ-3 传感器引脚定义

```
MQ-3模块 → STM32F407ZGT6
├─ VCC   → 5V（从开发板JP3的VCC5取电）
├─ GND   → GND（共地）
├─ AO    → PA5 (ADC1_CH5) 通过分压电路
└─ DO    → 可选（数字输出，本项目未使用）
```

### 1.2 ⚠️ 重要：电压分压电路

**MQ-3 输出电压是 0-5V，而 STM32 ADC 只能接受 0-3.3V！**

**必须添加分压电路：**

```
MQ-3 AO输出(0-5V)
    │
   ┌─┴─┐
   │15kΩ│  ← 优化后的分压电路
   └─┬─┘
    ├────→ PA5 (STM32 ADC输入, 0-3.3V)
   ┌─┴─┐
   │10kΩ│
   └─┬─┘
    │
   GND
```

**分压公式：** Vout = Vin × R2 / (R1 + R2) = 5V × 10kΩ / 25kΩ = 2.0V (安全)

### 1.3 完整硬件清单

| 序号 | 组件名称 | 数量 | 说明 |
|:---:|:---|:---:|:---|
| 1 | STM32F407ZGT6开发板 | 1 | 主控MCU |
| 2 | MQ-3酒精传感器模块 | 1 | 带模拟输出AO |

### 2.3 JSON数据格式

**旧格式（V2.0）：**
```json
{
  "counter": 123,
  "adc": 2048,
  "voltage": 1.65
}
```

**新格式（V3.0 - 支持MQ-3）：**
```json
{
  "counter": 123,
  "adc": 2048,
  "voltage": 1.65,
  "mq3_adc": 1850,
  "mq3_voltage": 1.35,
  "alcohol_ppm": 125.5,
  "sensor_status": 0
}
```

**字段说明：**
- `mq3_adc`: MQ-3传感器ADC原始值 (0-4095)
- `mq3_voltage`: MQ-3传感器电压值 (V)，经过分压还原
- `alcohol_ppm`: 酒精浓度 (ppm)
- `sensor_status`: 0=就绪, 1=预热中, 2=错误

## 🚀 三、部署步骤

### 步骤1：硬件连接

1. **断电操作：** 确保 STM32 和 MQ-3 都断电
2. **搭建分压电路：** 在面包板上连接 15kΩ 和 10kΩ 电阻
3. **连接传感器：**
   - MQ-3 VCC → JP3 的 VCC5 (引脚7或8)
   - MQ-3 GND → GND（与STM32共地）
   - MQ-3 AO → 分压电路输入
   - 分压电路输出 → STM32 PA5
4. **万用表测试：** 在接入STM32前，测量分压输出电压 < 3.3V

## 🧪 四、测试与校准

### 4.1 传感器预热

MQ-3 传感器需要预热才能正常工作：

- **首次使用：** 24-48 小时
- **长期未用：** 24 小时
- **代码中设置：** 3 分钟（仅用于快速测试）

**建议：** 在正式测试前，让传感器通电预热至少 30 分钟。

### 4.2 校准传感器

在 **清洁空气** 环境中校准：

```c
// 在 main.c 的初始化代码中添加：
MQ3_Calibrate();  // 会自动计算并更新R0基准值
```

### 4.3 功能测试

1. **基线测试：** 在清洁空气中，酒精浓度应接近 0 ppm
2. **响应测试：** 将酒精（如医用酒精）靠近传感器，观察浓度上升
3. **恢复测试：** 移除酒精源，浓度应逐渐下降

### 4.4 典型浓度参考

| 酒精来源 | 预期浓度 (ppm) |
|:---|:---|
| 清洁空气 | 0-10 |
| 轻微酒精蒸气 | 50-100 |
| 医用酒精棉球（近距离） | 200-500 |
| 饮酒后呼气 | 100-300 |

## 🔧 五、故障排查

### 问题1：传感器一直显示"预热中"

**原因：** 供电不足或传感器损坏

**解决方案：**
1. 确认 MQ-3 使用独立 5V 电源供电（不要从STM32取电）
2. 检查加热器是否发热（传感器会微热）
3. 修改代码跳过预热检查（仅测试用）：
   ```c
   mq3_config.is_preheated = true;  // 强制设为已预热
   ```

### 问题2：酒精浓度读数异常（过高或过低）

**原因：** 未校准或分压电路错误

**解决方案：**
1. 在清洁空气中运行 `MQ3_Calibrate()`
2. 检查分压电路阻值
3. 调整 `sensor_manager.c` 中的参数：
   ```c
   #define MQ3_R0_CLEAN_AIR  60.0f   // 根据实际情况调整
   #define VOLTAGE_DIVIDER_RATIO  1.5f  // 根据分压电路调整
   ```

### 问题3：ADC读数为0或4095

**原因：** 分压电路连接错误或传感器未供电

**解决方案：**
1. 用万用表测量 PA5 引脚电压，应在 0-3.3V 范围内
2. 检查 AO 引脚是否有输出（用万用表测量）
3. 确认 STM32 的 ADC 参考电压 VREF+ 已连接到 3.3V

### 问题4：数据无法存入数据库

**错误信息：** `数据库保存失败: unknown column 'mq3_adc'`

**解决方案：**
```bash
# 运行数据库迁移脚本
python backend/migrate_database.py
```

## 🎯 七、核心代码说明

### 7.1 MQ-3 浓度计算原理

```c
/* 步骤1：读取ADC原始值 (0-4095) */
uint16_t adc_raw = MQ3_ReadADC();

/* 步骤2：转换为电压 (0-3.3V) */
float voltage = (adc_raw / 4096.0f) * 3.3f;

/* 步骤3：还原实际传感器输出电压 (0-5V) */
voltage *= 1.5f;  // 根据分压比

/* 步骤4：计算传感器电阻 Rs */
float Rs = (5.0f - voltage) * RL / voltage;

/* 步骤5：计算 Rs/R0 比值 */
float rs_r0_ratio = Rs / R0;

/* 步骤6：根据数据手册曲线计算PPM */
// MQ-3对酒精的响应曲线：ppm = A * (Rs/R0)^B
float ppm = 0.4f * powf(rs_r0_ratio, -1.431f);
```

### 7.2 传感器状态机

```c
typedef enum {
    SENSOR_STATUS_OK = 0,         // 正常工作
    SENSOR_STATUS_PREHEATING,     // 预热中
    SENSOR_STATUS_ERROR,          // 错误
    SENSOR_STATUS_NOT_READY       // 未就绪
} SensorStatus_t;
```
