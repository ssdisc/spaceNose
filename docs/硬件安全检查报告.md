```c
{

    /* ADC转电压 */
    float voltage = (adc_raw / ADC_RESOLUTION) * ADC_VREF;  // ADC → PA5电压
    // voltage 范围：0 ~ 3.3V (实际最大约 2.0V)

    /* 如果使用了分压电路，需要还原真实电压 */
    voltage *= VOLTAGE_DIVIDER_RATIO;                    // PA5电压 → MQ-3电压
    // voltage 范围：0 ~ 5V

}
```

**场景3：极端情况（MQ-3 异常输出 5V）**

## 🚨 潜在风险点（已缓解）

### ⚠️ 风险1：首次上电瞬态

**描述：** 首次上电时，GPIO 可能处于未定义状态，短暂时间内可能有异常电流。

**缓解措施：**
```c
// stm32f4xx_hal_msp.c:117
GPIO_InitStruct.Mode = GPIO_MODE_ANALOG;  // ✅ 立即配置为模拟输入
GPIO_InitStruct.Pull = GPIO_NOPULL;       // ✅ 无上下拉，不驱动电流
```

**实际影响：** 🟢 **可忽略**（HAL 库会快速初始化 GPIO）

---

### ⚠️ 风险2：分压电路接反

**描述：** 如果用户错误地将 15kΩ 接地，10kΩ 接 MQ-3，会导致输出电压增大。

**最坏情况计算：**
```
错误接法：
MQ-3 AO (5V)
    │
   10kΩ (错误地接在上面)
    │
    ├──→ PA5
    │
   15kΩ
    │
   GND

输出电压 = 5V × 15kΩ / 25kΩ = 3.0V
```

**安全性：** ✅ **仍然安全！** 3.0V < 3.3V

**缓解措施：**
- 文档中有清晰的接线图
- 快速参考卡中标注了正确接法
- 即使接反，也不会损坏硬件

**实际影响：** 🟢 **低风险**（最多降低测量精度）

---

### ⚠️ 风险3：MQ-3 电源未连接

**描述：** 如果 MQ-3 的 VCC 未连接，传感器不工作。

**影响：**
- MQ-3 AO 可能输出不稳定电压或浮空
- PA5 可能读取到随机值

**缓解措施：**
```c
// 代码中会读取到低电压或异常值
// 不会损坏硬件，只是数据不准确
```

**实际影响：** 🟢 **无硬件风险**（只是功能失效）
